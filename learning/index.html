学习了浏览器缓存的相关知识

1、根据缓存的策略可以将缓存分为 强缓存 和 协商缓存。强缓存优先级高于协商缓存。（浏览器会先查找强缓存，然后查找协商缓存）
    1.1、强缓存：根据 Cache-Control（http1.1引入） 和 Expires（基本没用，保留它是为了兼容老版浏览器） 来判断强缓存是否失效。 Cache-Control 优先级高于 Expires。
    1.2、协商缓存（协商指客户端和服务器进行协商）：根据 Last-Modified / If-Modified-Since 和 ETag / If-None-Match 来判断缓存是否失效。 ETag / If-None-Match 优先级比 Last-Modified / If-Modified-Since 高。

2、根据缓存的位置可以将缓存分为 Service Worker(独立线程)、 Memory Cache(内存缓存)、 Disk Cache(磁盘缓存)、Push Cache(推送缓存)。
    2.1、Service Worker：它可以让我们自由控制缓存哪些文件、如何匹配缓存、如何读取缓存，并且缓存是持续性的。
    2.2、Memory Cache：一旦Tab页关闭，内存缓存就被释放了。速度要比磁盘缓存快。
    2.3、Disk Cache：磁盘缓存时效更长，存储量更大。
    2.4、Push Cache：HTTP2新增的缓存缓存。它只在会话（Session）中存在，一旦会话结束就被释放，并且缓存时间也很短暂，存活时间大概只有5分钟左右。

文件缓存的实际位置是受浏览器自身的机制控制的。

参考文档：深入理解浏览器的缓存机制、彻底理解浏览器的缓存机制

学习了V8垃圾回收的知识。

1、V8垃圾回收的两个主要部分是 新生代 和 老生代。
2、新生代使用 Scavenge算法 以空间换时间的方式交换from区间和to区间进行垃圾回收。
3、对象晋升：满足这两个条件之一的对象就会被移到老生代——a.已经经历过一次Scavenge算法；b.To空间内存使用率已经超过25%
4、老生代使用 标记清除 + 标记管理 的方案实现垃圾回收。

问：react里的Link标签和a标签有什么区别？
答：
1、Link标签如果绑定了onClick事件，就会默认组件跳转事件；
2：Link的跳转是使用hash跳转，不是页面跳转，只会更新对应的Router，不会刷新整个页面。
